#### Minor Observations (non-blocking)

1. **JSON `omitempty` on envelope fields** (`internal/cli/output.go` lines 11-12): The `Data` and `Error` fields use `json:"...,omitempty"`, which omits them when nil instead of rendering as `"data": null` / `"error": null`. The spec examples show explicit nulls. This is a cosmetic difference — agents parsing JSON will handle both — but could be tightened to match spec exactly by removing `omitempty`.

2. **`NETWORK_ERROR` code unused**: The code is defined but `mapConfigError` never returns it. Network errors from `http.Client.Do` fall through to the default case and get classified as `API_ERROR`. A future improvement could detect `net.Error` or timeout errors and map them to `NETWORK_ERROR`.

3. **Custom `bytesReader` in `applications.go`**: Lines 114-130 implement a custom `io.Reader` wrapper for `[]byte`. The stdlib `bytes.NewReader` already provides this functionality. Not a correctness issue but unnecessary code.
