## Instructions

Review the implemented code against specs. Check each of the following:

1. **API safety boundary**: no methods exist for stop, delete, update, create, or env var endpoints. Only healthcheck, version, list apps, get app, get logs, deploy, restart.
2. **Authentication**: Bearer token set on every request. Token never logged or printed.
3. **Error handling**: all HTTP status codes from 08-api-matrix section 5 handled (401, 404, 429 especially).
4. **Rate limit handling**: 429 response surfaces Retry-After header value.
5. **JSON envelope**: all commands produce `{ok, data, error}` structure per D6.
6. **Error codes**: match enumerated codes from 03-arch section 5.1.
7. **Permission check**: every agent command calls `enforcer.Check()` BEFORE making API call.
8. **Deploy endpoint**: maps to `/api/v1/deploy?uuid=...` per D5. Force flag passed correctly.
9. **Redeploy endpoint**: maps to `/api/v1/applications/{uuid}/restart` per D5.
10. **Logs endpoint**: passes tail parameter as query string.
11. **List command**: works without project config.
12. **Exit codes**: 0 on success, 1 on error.
13. **Timeouts**: 30s for regular calls per 03-arch section 4.2.

Output your verdict as PASS or FAIL with specific issues listed.
