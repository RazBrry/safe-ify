### Codex Findings

**Round 1 — OVERALL: FAIL (1, 2, 3, 4, 5)**

1. SPEC COMPLIANCE | FAIL | The CLI never emits `NETWORK_ERROR` even though the architecture spec defines that error code. Transport failures and API failures both surface as `API_ERROR`.
2. SECURITY | FAIL | `uuid` values are concatenated directly into Coolify request paths without escaping or format validation, leaving a path-manipulation vector.
3. CODE QUALITY | FAIL | Error handling is flattened too aggressively — transport failures and API failures both surfaced as `API_ERROR` instead of being classified at the appropriate layer.
4. TEST COVERAGE | FAIL | No CLI integration tests for `deploy`, `redeploy`, `logs`, `status`, or `list`, including permission-denied and full command JSON-output flows.
5. INTEGRATION FIT | FAIL | Permission resolution duplicated in `internal/config/runtime.go` and `internal/cli/agent.go` instead of reusing `internal/permissions.Enforcer`.

### Fix Round 1

**Items fixed: 1, 2, 3, 5** (item 4 is test coverage — deferred to a dedicated test task per task instructions)

**1 & 3 — NETWORK_ERROR classification (internal/coolify/types.go, client.go, cli/agent.go)**

- Added `NetworkError` struct to `internal/coolify/types.go` with `Cause error`, `Error()`, and `Unwrap()` methods.
- In `internal/coolify/client.go`, `doRequest` now returns `&NetworkError{Cause: err}` for all `httpClient.Do` failures, instead of a plain `fmt.Errorf` wrapper. These are transport-layer failures (connection refused, timeout, DNS) that occur before any HTTP response is received.
- Added `mapCoolifyError(err error) string` to `internal/cli/agent.go` that uses `errors.As` to detect `*NetworkError` → `NETWORK_ERROR`, falling back to `API_ERROR` for `*CoolifyError` and other errors.
- All 5 CLI agent commands (`deploy`, `redeploy`, `logs`, `status`, `list`) now call `mapCoolifyError(err)` instead of hardcoding `ErrCodeAPIError` for Coolify client errors.
- Updated `internal/coolify/client_test.go` `TestClient_NetworkError` to positively assert `*NetworkError` is returned.

**2 — UUID validation (internal/coolify/applications.go, deploy.go)**

- Added `uuidPattern` regexp (`^[0-9a-fA-F]{8}-[0-9a-fA-F]{4}-[0-9a-fA-F]{4}-[0-9a-fA-F]{4}-[0-9a-fA-F]{12}$`) and `validateUUID(uuid string) error` function in `applications.go` (same package, available to both files).
- `GetApplication`, `GetLogs`, and `Restart` now call `validateUUID` before constructing paths with UUID concatenation.
- `Deploy` also validates the UUID even though it uses a query parameter (defence-in-depth).
- Updated test UUIDs in `client_test.go` from short strings (`"hgkks00"`, `"missing-uuid-123"`) to valid UUID format (`"a1b2c3d4-e5f6-7890-abcd-ef1234567890"`, `"00000000-0000-0000-0000-000000000000"`).

**5 — Permission duplication (internal/cli/agent.go, deploy.go, redeploy.go, logs.go, status.go, list.go)**

- Removed `checkPermission(runtime, command)` function from `agent.go`.
- `resolveAgentConfig` signature changed from `(*RuntimeConfig, *Client, error)` to `(*RuntimeConfig, *Client, *permissions.Enforcer, error)`.
- The function now builds a `*permissions.Enforcer` via `permissions.NewEnforcer(*globalCfg, *projectCfg)` using the raw configs it already loads, ensuring the single authoritative deny-list logic in `permissions.Enforcer` is used.
- All 5 agent command files updated to destructure the 4-return value and call `enforcer.Check("command")` instead of `checkPermission(runtime, "command")`.
- `list.go` discards the runtime return value (`_`) since it only needs the enforcer and client.

### Fix Round 1 — Test Coverage

**Item fixed: 4** — CLI integration tests for all 5 agent commands

**File created:** `internal/cli/agent_commands_test.go`

**Approach:** Full Cobra command-tree integration tests using `net/http/httptest` to mock the Coolify API. Tests execute the real `rootCmd` with `--json`, `--config`, and `--project` flags wired to temporary files in `t.TempDir()`. `rootCmd.SilenceUsage = true` is set per-test to prevent Cobra from appending usage text to stdout after command errors, ensuring the JSON envelope can be parsed cleanly.
