## Instructions

1. Create `internal/cli/doctor.go`:
   - Register `doctor` command on root.
   - Run diagnostic checks in order:
     a. **Global config check**: file exists, permissions correct (0600). Print pass/fail.
     b. **Instances check**: at least one instance configured. Print count.
     c. **Connectivity check**: for each instance, call `client.Healthcheck()`. Print pass/fail per instance.
     d. **Version check**: call `client.Version()`. Print Coolify version.
     e. **Project config check**: `.safe-ify.yaml` found (with parent traversal). Print pass/fail. If not found, skip remaining project checks but don't fail entirely.
     f. **Instance reference check**: project's instance name exists in global config. Print pass/fail.
     g. **App UUID check**: call `client.GetApplication()` with project's app UUID. Print pass/fail with app name.
     h. **Permissions check**: resolve permissions, list allowed and denied commands.

   - After all checks, output CLAUDE.md markdown snippet per 04-cli section 5.1:
     ```markdown
     ## safe-ify (Coolify Safety Layer)

     Instance: {name} ({url})
     Application: {app_name} ({uuid})

     ### Available commands

     | Command | Status |
     |---------|--------|
     | `safe-ify deploy --json` | Allowed/Denied |
     ...

     ### Usage

     All commands support `--json` for structured output.
     ```

   - Exit code: 0 if all checks pass, 1 if any critical check fails.
   - Print the markdown to stdout so it can be piped: `safe-ify doctor >> CLAUDE.md`.

2. Run `go build ./...` and verify success.
