## Instructions

1. Create `internal/cli/output.go`:
   - Define JSON envelope types per 03-arch section 5.1:
     ```go
     type Response struct {
         OK    bool        `json:"ok"`
         Data  interface{} `json:"data,omitempty"`
         Error *ErrorInfo  `json:"error,omitempty"`
     }
     type ErrorInfo struct {
         Code    string `json:"code"`
         Message string `json:"message"`
     }
     ```
   - `OutputJSON(w io.Writer, resp Response)` -- marshals with indent, writes to writer.
   - `OutputError(w io.Writer, code, message string)` -- creates error response, outputs.
   - Error codes: `PERMISSION_DENIED`, `CONFIG_NOT_FOUND`, `INSTANCE_NOT_FOUND`, `API_ERROR`, `NETWORK_ERROR`, `CONFIG_INSECURE`.

2. Create shared startup helper in `internal/cli/agent.go`:
   - `resolveAgentConfig(cmd *cobra.Command) (*config.RuntimeConfig, *coolify.Client, error)`:
     a. Find and load project config (with parent traversal).
     b. Load global config.
     c. Create RuntimeConfig via `config.ResolveRuntime()`.
     d. Create Coolify client.
     e. Return all three or error.
   - Implement `config.ResolveRuntime()` in config package if not already present (per 04-config-permissions section 3).

3. Create `internal/cli/deploy.go`:
   - Register `deploy` command. Flags: `--json`, `--force`.
   - Resolve config, check permission ("deploy"), call `client.Deploy()`, output result.

4. Create `internal/cli/redeploy.go`:
   - Register `redeploy` command. Flags: `--json`.
   - Resolve config, check permission ("redeploy"), call `client.Restart()`, output result.

5. Create `internal/cli/logs.go`:
   - Register `logs` command. Flags: `--json`, `--tail` (int, default 100).
   - Resolve config, check permission ("logs"), call `client.GetLogs()`, output result.

6. Create `internal/cli/status.go`:
   - Register `status` command. Flags: `--json`.
   - Resolve config, check permission ("status"), call `client.GetApplication()`, extract status fields, output.

7. Create `internal/cli/list.go`:
   - Register `list` command. Flags: `--json`.
   - `list` follows the same startup sequence as all other agent commands (per D12):
     a. Call `resolveAgentConfig(cmd)` to load project config, global config, and create the Coolify client.
     b. If no project config found, error: "No project config found. Run `safe-ify init` first." (exit code 1, or JSON error with code `CONFIG_NOT_FOUND` if `--json`).
     c. Check permission by calling `enforcer.Check("list")` BEFORE calling the API. If denied, return `PERMISSION_DENIED` error.
     d. Only if permission check passes: call `client.ListApplications()`, output result.

8. Each command: on permission denied, output error JSON (if `--json`) or text error, exit code 1.

9. Run `go build ./...` and verify success.
