## Instructions

1. Create `internal/tui/styles.go`:
   - Define lipgloss styles for success, error, info, and table output.
2. Create `internal/tui/forms.go`:
   - `AuthAddForm()` -- returns a `huh.Form` with fields: instance name (text, validated: alphanumeric + hyphens), Coolify URL (text, validated: valid URL), API token (text, masked).
   - `AuthRemoveForm(instances []string)` -- returns `huh.Select` for instance picker + `huh.Confirm` for deletion.
3. Create `internal/cli/auth.go`:
   - Register `auth` parent command on root.
   - `auth add` subcommand:
     a. Run TUI form from `tui.AuthAddForm()`.
     b. Validate token by making HTTP GET to `{url}/api/v1/healthcheck` with Bearer token (use simple `net/http` call; full Coolify client is T11).
     c. On success: load global config (or create empty), add instance, save.
     d. On failure: display error, exit.
     e. Handle existing instance name: prompt to overwrite or rename.
   - `auth remove` subcommand:
     a. Load global config.
     b. If no instances, print message and exit.
     c. Run TUI picker + confirm.
     d. Remove instance, save config.
   - `auth list` subcommand:
     a. Load global config.
     b. Print table: Name, URL, Token (masked: first 4 chars + `****`).
     c. Use lipgloss for table formatting.
4. Run `go mod tidy`.
5. Verify `go build ./...` succeeds.
