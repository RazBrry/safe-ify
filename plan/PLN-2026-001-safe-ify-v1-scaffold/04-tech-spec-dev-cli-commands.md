# PLN-2026-001 Tech Spec Developer: CLI Commands

## 1. Root Command

```
safe-ify [command] [flags]
```

Global flags:
- `--json` -- JSON output (agent-facing commands only)
- `--config` -- override global config path (default: `~/.config/safe-ify/config.yaml`)
- `--project` -- override project config path (default: `.safe-ify.yaml`)
- `--version` -- print version and exit

Root command with no subcommand prints help text.

## 2. Auth Commands

### 2.1 `safe-ify auth add`

**Role:** Human-facing, interactive TUI.

Flow:
1. Display `huh.NewForm()` with fields:
   - Instance name (text input, required, validated: alphanumeric + hyphens)
   - Coolify URL (text input, required, validated: valid URL)
   - API token (text input, required, masked in display)
2. Validate by calling Coolify `GET /api/v1/healthcheck` with provided token.
3. On success: append instance to global config, save file.
4. On failure: show error, offer retry.

**Error cases:**
- Instance name already exists: prompt to overwrite or rename.
- URL unreachable: show network error.
- Token invalid (401): show auth error.

### 2.2 `safe-ify auth remove`

**Role:** Human-facing, interactive TUI.

Flow:
1. Load global config.
2. If no instances: show message and exit.
3. Display `huh.NewSelect()` with instance names.
4. Confirm removal with `huh.NewConfirm()`.
5. Remove from config, save file.

### 2.3 `safe-ify auth list`

**Role:** Human-facing, non-interactive.

Output: table with columns: Name, URL, Token (masked: first 4 chars + `****`).

Uses lipgloss for table formatting.

## 3. Init Command

### 3.1 `safe-ify init`

**Role:** Human-facing, interactive TUI.

Flow:
1. Load global config.
2. If no instances: error, suggest `safe-ify auth add`.
3. Display `huh.NewSelect()` to choose instance.
4. Call Coolify API to list applications for that instance.
5. Display `huh.NewSelect()` to choose application (shows name + UUID).
6. Display permission configuration:
   - `huh.NewMultiSelect()` with all agent commands: deploy, redeploy, logs, status, list.
   - Selected commands are DENIED for this project.
7. Write `.safe-ify.yaml` to current directory.

**Generated `.safe-ify.yaml`:**

```yaml
# Generated by safe-ify init
instance: my-coolify
app_uuid: "hgkks00"
permissions:
  deny: []
```

## 4. Agent Commands (Non-Interactive)

All agent commands share this startup sequence:
1. Load project config (`.safe-ify.yaml`), searching current dir and parents.
2. Load global config.
3. Resolve runtime config (merge permissions).
4. Check permission for this command.
5. If denied: return error (exit code 1).
6. Execute API call.
7. Write audit log entry.
8. Output result (JSON if `--json`, text otherwise).

### 4.1 `safe-ify deploy`

**Flags:** `--json`, `--force`

**API call:** `GET /api/v1/deploy?uuid={app_uuid}` (add `&force=true` if `--force`).

**Response mapping:**
```json
{
  "ok": true,
  "data": {
    "message": "Deployment queued.",
    "deployment_uuid": "dep-uuid-123"
  }
}
```

### 4.2 `safe-ify redeploy`

**Flags:** `--json`

**API call:** `GET /api/v1/applications/{app_uuid}/restart`

**Response mapping:**
```json
{
  "ok": true,
  "data": {
    "message": "Restart triggered."
  }
}
```

### 4.3 `safe-ify logs`

**Flags:** `--json`, `--tail N` (default: 100)

**API call:** `GET /api/v1/applications/{app_uuid}/logs?tail={N}`

**Response mapping:**
```json
{
  "ok": true,
  "data": {
    "lines": ["log line 1", "log line 2"],
    "count": 2
  }
}
```

### 4.4 `safe-ify status`

**Flags:** `--json`

**API call:** `GET /api/v1/applications/{app_uuid}`

**Response mapping:** Extract status-relevant fields:
```json
{
  "ok": true,
  "data": {
    "uuid": "app-uuid",
    "name": "my-app",
    "status": "running",
    "fqdn": "https://app.example.com",
    "last_deployment": "2026-03-10T12:00:00Z"
  }
}
```

### 4.5 `safe-ify list`

**Flags:** `--json`

**API call:** `GET /api/v1/applications`

**Response mapping:**
```json
{
  "ok": true,
  "data": {
    "applications": [
      {
        "uuid": "app-uuid",
        "name": "my-app",
        "status": "running",
        "fqdn": "https://app.example.com"
      }
    ],
    "count": 1
  }
}
```

Note: `list` does not require a project config. It uses the global config's first instance or requires `--instance` flag.

## 5. Doctor Command

### 5.1 `safe-ify doctor`

**Role:** Utility, non-interactive.

Checks performed:
1. Global config exists and has correct permissions (0600).
2. At least one instance is configured.
3. Instance is reachable (healthcheck).
4. Project config exists (`.safe-ify.yaml`).
5. Referenced instance exists in global config.
6. App UUID is valid (API call to get application).
7. Resolved permissions computed.

Output: markdown block suitable for CLAUDE.md:

```markdown
## safe-ify (Coolify Safety Layer)

Instance: my-coolify (https://coolify.example.com)
Application: my-app (hgkks00)

### Available commands

| Command | Status |
|---------|--------|
| `safe-ify deploy --json` | Allowed |
| `safe-ify redeploy --json` | Allowed |
| `safe-ify logs --json --tail N` | Allowed |
| `safe-ify status --json` | Allowed |
| `safe-ify list --json` | Denied |

### Usage

All commands support `--json` for structured output.
```

Exit code: 0 if all checks pass, 1 if any fail (with details printed).
