## Instructions

1. Read all files in the Read list.
2. Verify the implementation against the spec. Check each item:

   **Config types:**
   - `AppConfig` struct has correct fields and yaml tags
   - `ProjectConfig.Apps` has correct type and yaml tag with `omitempty`
   - `RuntimeConfig.AppName` field exists

   **Loading (project.go):**
   - Legacy format detection: checks `AppUUID != ""` and `len(Apps) > 0`
   - Both-format rejection per D5
   - Legacy normalization: key "default", UUID copied, `AppUUID` cleared
   - App name validation: alphanumeric + hyphens regex
   - App UUID validation: non-empty
   - App deny list validation via `permissions.ValidateDenyList`

   **Runtime resolution:**
   - Signature includes `appName string`
   - Single-app default behavior
   - `AppAmbiguousError` for multi-app without name
   - `AppNotFoundError` for unknown name
   - Three-layer deny merge: global + project + app
   - `AppName` set on returned `RuntimeConfig`

   **Errors:**
   - `AppNotFoundError` and `AppAmbiguousError` exist with correct fields and `Error()` methods

   **Permissions:**
   - `NewEnforcer` has third `appDeny []string` parameter
   - App deny applied after project deny
   - `deniedBy` reports "app" correctly
   - Existing callers use `[]string{}` as placeholder

3. Check for anti-patterns:
   - No silent data loss
   - No panic paths
   - Error messages are user-friendly
   - No hardcoded values that should be constants

4. Output your verdict as PASS or FAIL.
