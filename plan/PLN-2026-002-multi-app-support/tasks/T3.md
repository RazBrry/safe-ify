#### Non-blocking Observations

1. **Duplicate deny list validator:** `project.go` defines its own `validateDenyList` rather than calling `permissions.ValidateDenyList`. Both are functionally identical. The local copy likely avoids an import cycle concern. Acceptable.
2. **Stale comment in `permissions/types.go`:** `PermissionDeniedError.DeniedBy` comment says `"global" or "project"` but now also supports `"app"`. Should be updated in a future task.
3. **`doctor.go` uses `projectCfg.AppUUID` directly (line 212):** After `LoadProject` normalization, `AppUUID` is always empty. The doctor command's app UUID check (7/8) will pass an empty string to `GetApplication`. This is a pre-existing issue; doctor multi-app support is scoped to S2 per CONTEXT.md.
4. **`agent.go` hardcodes empty appName and appDeny:** Lines 68 and 75 use `""` and `[]string{}` as S2 placeholders. Comments document this correctly.
