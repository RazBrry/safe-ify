## Instructions

### Part 1: --app flag and agent.go changes

1. **Update `internal/cli/root.go`:**
   - Add `appName string` variable alongside existing `jsonOutput`, `configPath`, `projectPath`
   - Register persistent flag: `rootCmd.PersistentFlags().StringVar(&appName, "app", "", "Target app name (required when multiple apps configured)")`

2. **Update `internal/cli/output.go`:**
   - Add error code constants:
     ```go
     ErrCodeAppNotFound  = "APP_NOT_FOUND"
     ErrCodeAppAmbiguous = "APP_AMBIGUOUS"
     ```

3. **Update `internal/cli/agent.go`:**
   - In `resolveAgentConfig`: change signature to `resolveAgentConfig(cmd *cobra.Command, appRequired bool)`.
   - Read the `--app` flag: `appName, _ := cmd.Root().PersistentFlags().GetString("app")`
   - Change the `ResolveRuntime` call to pass `appName`: `config.ResolveRuntime(globalCfg, projectCfg, appName)`
   - When `appRequired` is false and `ResolveRuntime` returns `*config.AppAmbiguousError`, set `cfg.AppUUID = ""` and `cfg.AppName = ""` instead of returning an error. This enables the `list` command to work without `--app` (D6).
   - When `appRequired` is true, propagate `AppAmbiguousError` and `AppNotFoundError` as errors.
   - Update `mapConfigError` to handle new error types:
     ```go
     case *config.AppNotFoundError:
         return ErrCodeAppNotFound
     case *config.AppAmbiguousError:
         return ErrCodeAppAmbiguous
     ```
   - Update `runAgentCommand` to accept `appRequired bool` and pass it to `resolveAgentConfig`.
   - In `runAgentCommand`, set `entry.AppName = cfg.AppName` on the audit entry.
   - Update all command callers: `deploy`, `redeploy`, `logs`, `status` pass `appRequired: true`. `list` passes `appRequired: false`.

### Part 2: Init command multi-app support

4. **Add `InitAppNameForm` to `internal/tui/forms.go`:**
   - Function: `InitAppNameForm(defaultName string, existingNames []string, name *string) *huh.Form`
   - Single input field for app name
   - Validation: non-empty, matches `^[a-zA-Z0-9][a-zA-Z0-9-]*$`, not in `existingNames`
   - Default value set to `defaultName`

5. **Update `runInit` in `internal/cli/init.go`:**

   **Case A: No existing config (new project)**
   - After the application picker, add the app name prompt:
     - Compute a default name by sanitizing the Coolify app name (lowercase, replace spaces with hyphens, strip invalid chars)
     - Run `tui.InitAppNameForm(defaultName, nil, &appNameValue)`
   - Build `ProjectConfig` in the new multi-app format:
     ```go
     projectCfg := &config.ProjectConfig{
         Instance: selectedInstance,
         Apps: map[string]config.AppConfig{
             appNameValue: {
                 UUID:        selectedAppUUID,
                 Permissions: config.PermissionConfig{Deny: denyList},
             },
         },
         Permissions: config.PermissionConfig{Deny: []string{}},
     }
     ```
   - Remove the old `AppUUID` assignment

   **Case B: Existing config found**
   - Before the instance picker, check if `.safe-ify.yaml` exists in cwd
   - If it exists, load it (this auto-normalizes legacy format per D2)
   - Prompt: "Add another app to this project?" (use `huh.NewConfirm`)
   - If no: print message and exit
   - If yes: use the existing instance (no instance picker needed), fetch applications, application picker, app name prompt (pass existing app names to prevent duplicates), deny list picker, add new app to `Apps` map, save

6. **Update success summary** to show the app name alongside the app UUID.

### Part 3: Doctor command multi-app support

7. **Update check 7 (App UUID check) in `runDoctor`:**
   - Instead of calling `GetApplication` once with `projectCfg.AppUUID`, loop over `projectCfg.Apps`:
     ```go
     for appName, appCfg := range projectCfg.Apps {
         app, appErr := client.GetApplication(ctx, appCfg.UUID)
         // report PASS/FAIL per app
     }
     ```

8. **Update check 8 (Permissions check):**
   - Loop over `projectCfg.Apps`, build per-app enforcer:
     ```go
     for appName, appCfg := range projectCfg.Apps {
         enforcer := permissions.NewEnforcer(*globalCfg, *projectCfg, appCfg.Permissions.Deny)
         // report allowed/denied per app
     }
     ```

9. **Update CLAUDE.md snippet output:**
   - List each app with command permissions
   - Include `--app <name>` in the command column when multiple apps configured
   - If only one app, omit `--app` from the commands (single-app default behavior)

10. **Remove all references to `projectCfg.AppUUID`** in doctor.go. Use `projectCfg.Apps` exclusively.

### Part 4: Audit entry update

11. **Update `internal/audit/types.go`:**
    - Add `AppName string` field to `Entry` struct
    - Update `Entry.String()` to include AppName:
      ```
      Format: YYYY-MM-DDTHH:MM:SSZ | command | app_name | app_uuid | instance | result | duration_ms
      ```

12. **Verify `go build ./...` compiles.**
