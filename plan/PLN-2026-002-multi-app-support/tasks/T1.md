**Plan:** PLN-2026-002
**Slice:** S1

## Context

This task implements the core config layer changes for multi-app support. Per D1, apps are stored as a YAML map. Per D2, legacy single-app configs are auto-detected and normalized internally. Per D5, configs with both `app_uuid` and `apps` are rejected. The runtime resolution (per D3) now accepts an app name and performs three-layer deny merging.

## Instructions

1. **Update `internal/config/types.go`:**
   - Add `AppConfig` struct with `UUID string` and `Permissions PermissionConfig` (yaml tags: `uuid`, `permissions`)
   - Add `Apps map[string]AppConfig` field to `ProjectConfig` with yaml tag `apps,omitempty`
   - Mark existing `AppUUID` field with yaml tag `app_uuid,omitempty` (it already has `app_uuid`)
   - Add `AppName string` field to `RuntimeConfig`

2. **Update `internal/config/errors.go`:**
   - Add `AppNotFoundError` struct with fields `Name string` and `AvailableApps []string`. Implement `Error()`.
   - Add `AppAmbiguousError` struct with field `AvailableApps []string`. Implement `Error()`.

3. **Update `internal/config/project.go` — `LoadProject` function:**
   - After unmarshaling, detect format:
     - If both `AppUUID != ""` and `len(Apps) > 0`: return error "config has both 'app_uuid' and 'apps'; use only one format"
     - If `AppUUID != ""` and `Apps` is nil/empty: normalize to `Apps = map[string]AppConfig{"default": {UUID: AppUUID, Permissions: PermissionConfig{Deny: []string{}}}}`, then clear `AppUUID`. Note: the app gets an empty deny list, NOT a copy of the project-level deny. The project-level `cfg.Permissions` remains at the project level and is applied separately during `ResolveRuntime` (per 04-tech-spec-dev-config-migration.md step 3).
     - If `Apps` is not nil and not empty: validate each app (non-empty UUID, valid app name pattern `^[a-zA-Z0-9][a-zA-Z0-9-]*$`)
     - If both are empty: return error "no apps configured"
   - Validate app-level deny lists using `permissions.ValidateDenyList` (import the package)
   - Remove the existing `app_uuid` required validation (replaced by the above logic)

4. **Update `internal/config/project.go` — `SaveProject` function:**
   - No changes needed; the existing YAML marshal handles the new fields. But ensure `AppUUID` is not written (it has `omitempty` tag and is cleared during load).

5. **Update `internal/config/runtime.go` — `ResolveRuntime` function:**
   - Change signature to `ResolveRuntime(global *GlobalConfig, project *ProjectConfig, appName string) (*RuntimeConfig, error)`
   - App lookup logic:
     - If `appName == ""` and `len(project.Apps) == 1`: use the only app, set its name
     - If `appName == ""` and `len(project.Apps) > 1`: return `&AppAmbiguousError{AvailableApps: <sorted keys>}`
     - If `appName != ""`: look up in `project.Apps`, return `&AppNotFoundError{...}` if not found
   - Deny list merge: union of `global.Defaults.Permissions.Deny` + `project.Permissions.Deny` + `selectedApp.Permissions.Deny`
   - Set `RuntimeConfig.AppName` to the resolved app name
   - Set `RuntimeConfig.AppUUID` to the resolved app's UUID
