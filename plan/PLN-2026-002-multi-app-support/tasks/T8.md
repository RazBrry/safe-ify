## Instructions

1. **Create/update `internal/audit/types_test.go`** (or `entry_test.go`):
   - `TestEntry_StringWithAppName`: Create Entry with AppName="api", verify String() output includes "api" in the correct pipe-delimited position (between command and app_uuid).
   - `TestEntry_StringEmptyAppName`: Create Entry with empty AppName, verify String() handles it gracefully (empty field between pipes).

2. **Create/update CLI integration tests** (in appropriate `*_test.go` files under `internal/cli/`):

   Follow existing test patterns in the codebase. These tests may need test fixtures (temp config files with multi-app format).

   - `TestMapConfigError_AppNotFound`: Call `mapConfigError` with `&config.AppNotFoundError{Name: "unknown", AvailableApps: []string{"api"}}`, verify it returns `ErrCodeAppNotFound` (`"APP_NOT_FOUND"`).
   - `TestMapConfigError_AppAmbiguous`: Call `mapConfigError` with `&config.AppAmbiguousError{AvailableApps: []string{"api", "web"}}`, verify it returns `ErrCodeAppAmbiguous` (`"APP_AMBIGUOUS"`).

   Using Cobra test patterns (command execution in-process with captured output):
   - `TestDeployCmd_WithAppFlag`: Multi-app config with `--app=api`, verify correct UUID used in the request.
   - `TestDeployCmd_AmbiguousApp`: Multi-app config without `--app` flag, verify JSON error response contains `APP_AMBIGUOUS` code.
   - `TestDeployCmd_AppNotFound`: Multi-app config with `--app=unknown`, verify JSON error response contains `APP_NOT_FOUND` code.
   - `TestListCmd_NoAppRequired`: Multi-app config without `--app` flag, verify list command does NOT error with `APP_AMBIGUOUS`.
   - `TestDoctorCmd_MultiApp`: Multi-app config, verify doctor output includes all configured app names.

3. **Run tests:**
   ```bash
   go test ./internal/audit/
   go test ./internal/cli/
   ```

4. **Run full suite:**
   ```bash
   go test ./...
   ```
