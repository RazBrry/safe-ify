# T7 — Implement init command multi-app support

**Plan:** PLN-2026-002
**Slice:** S2
**Role:** Implementer

## Context

The `init` command must support both new project setup (single app in new format) and adding apps to existing projects. Per D2, existing legacy configs are auto-normalized when loaded. The init flow adds an app name prompt and saves in the new multi-app format.

## Instructions

1. **Add `InitAppNameForm` to `internal/tui/forms.go`:**
   - Function: `InitAppNameForm(defaultName string, existingNames []string, name *string) *huh.Form`
   - Single input field for app name
   - Validation:
     - Non-empty
     - Matches `^[a-zA-Z0-9][a-zA-Z0-9-]*$` (reuse `instanceNameRE` or define a similar regex)
     - Not in `existingNames` (error: "app name already exists")
   - Default value set to `defaultName`

2. **Update `runInit` in `internal/cli/init.go`:**

   **Case A: No existing config (new project)**
   - After the application picker (step e), add the app name prompt:
     - Compute a default name by sanitizing the Coolify app name (lowercase, replace spaces with hyphens, strip invalid chars)
     - Run `tui.InitAppNameForm(defaultName, nil, &appNameValue)`
   - After the deny list picker (step f), build `ProjectConfig` in the new multi-app format:
     ```go
     projectCfg := &config.ProjectConfig{
         Instance: selectedInstance,
         Apps: map[string]config.AppConfig{
             appNameValue: {
                 UUID:        selectedAppUUID,
                 Permissions: config.PermissionConfig{Deny: denyList},
             },
         },
         Permissions: config.PermissionConfig{Deny: []string{}},
     }
     ```
   - Remove the old `AppUUID` assignment

   **Case B: Existing config found**
   - Before step b (instance picker), check if `.safe-ify.yaml` exists in cwd
   - If it exists, load it (this auto-normalizes legacy format per D2)
   - Prompt: "Add another app to this project?" (use `huh.NewConfirm`)
   - If no: print message and exit
   - If yes:
     - Use the existing instance (no instance picker needed)
     - Fetch applications from the existing instance
     - Application picker
     - App name prompt (pass existing app names to prevent duplicates)
     - Deny list picker
     - Add new app to the `Apps` map
     - Save

3. **Update success summary** to show the app name alongside the app UUID.
