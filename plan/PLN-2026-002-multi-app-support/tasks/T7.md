#### Minor observations (non-blocking)

1. **Doctor `--app` flag display insertion** (`doctor.go:272-274`): The replacement logic inserts `--app <name>` before the closing backtick. For example, `` `safe-ify deploy --json` `` becomes `` `safe-ify deploy --json --app frontend` ``. The spec says "list each app with `--app <name>`" which is satisfied, though the flag appears after `--json` rather than before. This is cosmetic only and functionally correct for Cobra (flag order does not matter).

2. **Hardcoded AllowedCmds in partial runtime** (`agent.go:85`): When `appRequired` is false and `AppAmbiguousError` is caught, the fallback `AllowedCmds` map is hardcoded. This is acceptable since the `list` command is the only consumer and it does its own permission check. However, if future non-app-required commands are added, this hardcoded map could be a maintenance burden. Non-blocking.

3. **`_ = ambig` on line `agent.go:76`**: This variable assignment exists to suppress an unused-variable warning. Clean but could simply use `_` in the type assertion. Non-blocking style nit.

All spec requirements from `04-tech-spec-dev-cli-commands.md` and decisions D4, D6 are correctly implemented. No blocking issues found. Error handling is consistent and user-facing messages are clear. The implementation follows the allowlist-only and fail-safe design principles.
